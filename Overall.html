2. Overall Description


  • 2.1 Product Perspective

To-let is a GPS-Web-based house renting solution, where any needful user can easily find their rentable house/flat in desired location. There are other websites which are already providing house rent solutions, so how To-let is differed to them. To-let give the users to see exactly where the desired home/flat is, and with a map view, he /she can easily find the destination details. It also provide a mobile application by which, a user can just open an app, through camera, gyroscope, GPS sensors and with the help of third-party API, he/she can easily see that, if there is any rentable home near him/her and get all the important details. Which makes User life easier when he/she is looking for a home for rent. It also uses data of renowned sites like Bikroy or bdproperty etc. by web scrapping. Users can also give ads to give rent their houses/flats etc.

The following information that are included in To-let:
1) Customer Detail: This system will have customer detail like: name, phone, email, address etc., who is going to give ads of their home and also those customer who comes here for finding homes, so that if the user tick some product (flat/house) as favorite, he/she can easily found that when he come back again and also system can send some suggestion according to his/her need. It is not mandatory or need for a user who just looking for houses for rent to give his/her information.
2) Map View: The user can see or can find his/her desired location houses in a map (Google map) which integrated with the help of third-party API.
3) Houses List: Users will have a list of houses according to his/her search result, to find his/her rentable home easily.

Product Functions

At first User have to search To-let URL or get the link from somewhere. When user click on the link, user can see a welcome page. Where he/she can search/ browse to find rentable/sellable houses/flats/apartments etc. If the user wants to give ads for rent, he/she need to sign up To-let. The user will have a user panel where he/she can add or remove his/her ads. The user can log out anytime. Here, the admin can also sign in but can not register. Admin will also have a panel for him/her.

  • 2.2 User Classes and Characteristics

There are two types of user Classes in To-let. First one is admin and the next one is user. There are two types of user. One type of user want to give ads to sell or rent their property and other type of user want to find a home for accommodation. User has no privilege to access in the database, but admin does. Both user and admin can sign in, but sign up can only done by users. User can search home/apartment etc. and get their desired information that they are looking for.

  • 2.4 Design and Implementation Constraints

1.Admin needs to check and update the database for new houses detail list by manually or by the help of integrated API.
2.Users devices having the system (Website/app) must be connected via WIFI or Mobile Data.
3.Users devices must have a GPS system in built. (For android phones for searching in Argument Reality, user devices must have gyroscope, AI functionality).
4.User devices should have sufficient memories.
5.Users must search within Dhaka region to avoid uncertain issues, because for now it will work only in this region.
