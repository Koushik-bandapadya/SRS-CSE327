1. Introduction

1.1 Purpose

The purpose of the Software Requirement Specification (SRS) is to specify the functionality, performance and interface requirements of the software project. All of the required features of our To-let project will be expressed in this document. The formal release version of the project will meet every requirement listed in this document.

1.2 Document Conventions

Following conventions are given below.

DB Database
RDB Relational Database
ER Entity Relationship
AI Artificial Intelligence
VR Virtual reality

1.3 Intended Audience and Reading Suggestions

This project is a prototype for any person who is finding a house for rent in a particular area. The person can check available house in the map and get all the other details about the house .

1.4 Product Scope

To-let is a real time location-based renting houses system. The application offers various features like - price list ,price range , location-based deals ,trending house-deal etc. These services are conveniently grouped and developed for use on both android device and Computer. All the operations can be performed when an internet connection is available.
