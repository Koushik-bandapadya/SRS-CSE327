4. System Features

4.1.1 Description and Priority

There will be a list of houses which can be rent .

4.1.2 Stimulus/Response Sequences

Actors: Admin User
Use case:
1. Welcome page
2. Search House
3. Select House
4. Sign in
5. Sign out
6. Admin panel
7. Client panel
8. View House List
9. House Ads
10. Map View

4.1.3 Functional Requirements

Expanded use case 1:

Use Case Name: “Got to Welcome Page”

Use case Type: Primary

Actor(s): Admin, User

Actor Actions:

1. Go to To-let home page URL via a web browser (For android app, just open the app.

System Responses

2. Browser/android app serves the landing/home page.

Alternative course of event:

N/A




Expanded use case 2:

Use Case Name: “Got to Search House”

Use case Type: Primary

Actor(s):User

Actor Actions:

1. In the homepage user can click on type in search section and click the search button.

System Responses

2. System respond with search result of rentable houses.

Alternative course of event:

1.1) If the search is not matched with any detail of database house list, then system show “No found houses for rent” message, with a button to go back to the home page.




Expanded use case 3:

Use Case Name: “Select House”

Use case Type: Secondary

Actor(s):User

Actor Actions:

1. In search result or any other ads in the homepage, actor can select houses

3. Actor can click the house name from the house list.

System Responses

2. System responds with Select houses.

4.System responds with all the information of the house list.

Alternative course of event:

N/A




Expanded use case 4:

Use Case Name: “Sign up” Use case Type: Primary

Use case Type: Primary

Actor(s): Admin, User

Actor Actions:

1. In the homepage there user can click on the sign up section and can sign up if he/she already haven’t have any account by fill up the from given.

System Responses

2. System will store new user sign up data.

Alternative course of event:

1.1) If user put wrong information or put easy password system ,then system show “Password must be strong” .




Expanded use case 5:

Use Case Name: “Sign in”
Use case Type: Primary
Actor(s): Admin, User

Actor Actions:

1. User/admin can click on the sign in section and can sign in by given email/name and password data

System Responses

2. System will verify sign in data and if it is match with given user data, then system will allow the user/admin to sign in.

Alternative course of event:

2.1) If user put invalid sign in information, then system will show “Invalid sign in information” message, and got back to sign in form




Expanded use case 6:

Use Case Name: “Admin panel”
Use case Type: Primary
Actor(s): Admin

Actor Actions:

1. After sign in, admin panel will open for admin.

System Responses

2. System will allow admin to access all the functionality admin have.

Alternative course of event:

N/A




Expanded use case 7:

Use Case Name: “Client panel”
Use case Type: Primary
Actor(s): User

Actor Actions:

1. After valid sign in, user panel will open for user.

System Responses

2. System will allow user to access all the functionality user have.

Alternative course of event:

N/A




Expanded use case 8:

Use Case Name: “View House List”
Use case Type: Primary
Actor(s): User, Admin

Actor Actions:

1. In the user/admin panel one particular section will be for all available rentable house list.

System Responses

2. System will show all the available rentable houses detail.

Alternative course of event:

2.1) If system can’t find any rentable house then system will show “No house” message, and got back to view house section form.




Expanded use case 9:

Use Case Name: “House Ads”
Use case Type: Secondary
Actor(s): User

Actor Actions:

1. In House ads section, user can see rentable houses ads.

System Responses

2. System will redirect to the page where all the available rentable houses adds are in listed.

Alternative course of event:

N/A




Expanded use case 10:

Use Case Name: “Map View”
Use case Type: Primary
Actor(s): User, Admin

Actor Actions:

1. In the map view section admin and user both can see real time house map location.

System Responses

2. System will show all the available rentable houses in the map.

Alternative course of event:

2.1) If system cannot find any rentable house then system will show “No rentable house in this area” message, and got back to map view section form.
